# ZachetOS / cdm-os

Учебная мини-операционная система для процессора **CdM-16** в **Logisim**.
Проект демонстрирует базовые идеи ОС: обработку прерываний, терминальный ввод/вывод, запуск пользовательских программ через таблицу программ, библиотечные функции и подготовку к диспетчеризации задач.

## Кратко о проекте

**ZachetOS** — это простая OS-like среда, написанная на ассемблере CdM-16. После запуска система выводит приветствие, открывает терминал и ждёт команды пользователя. Команды ищутся в таблице программ, после чего управление передаётся соответствующему коду программы.

Сейчас реализованы:

- загрузка ядра через таблицу векторов прерываний;
- обработчик клавиатурного прерывания;
- терминальный вывод через memory-mapped I/O;
- shell loop с приглашением `>`;
- таблица программ в памяти;
- запуск программ по имени из терминала;
- библиотечная функция ввода строки, аналог `gets()` / `fgets()`;
- перевод строки в число (`atoi`);
- перевод числа в строку (`itoa`);
- программа `input` для проверки пользовательского ввода;
- программа `calc`, которая получает числа от пользователя и складывает их;
- таймер пребывания / основа для учёта времени работы системы;
- подготовка к следующему этапу — диспетчеру задач.

## Команды терминала

После запуска ОС пользователь работает через терминал.

Доступные команды:

```text
prog1
prog2
ls
input
calc
```

### `prog1`

Тестовая программа. Проверяет запуск пользовательской программы через таблицу программ.

### `prog2`

Вторая тестовая программа. Нужна для проверки того, что ОС может хранить и запускать несколько программ.

### `ls`

Показывает список доступных программ.

### `input`

Проверяет библиотечный ввод строки.

Пример работы:

```text
>input
Enter text: hello
You typed: hello
>
```

Эта программа показывает, что пользовательская программа может сама запросить строку у пользователя, а не только получать команды из shell.

### `calc`

Простой калькулятор. Программа получает два числа от пользователя, переводит строки в числа, складывает их и выводит результат.

Пример работы:

```text
>calc
a: 2
b: 3
result: 5
>
```

Калькулятор демонстрирует полноценную цепочку:

1. получить строку от пользователя;
2. перевести строку в число;
3. выполнить арифметику;
4. перевести результат обратно в строку;
5. вывести результат в терминал.

## Структура проекта

Основные файлы проекта:

```text
kernel_main.asm      — таблица прерываний, обработчик клавиатуры, запуск ядра, выполнение команд
kernel_drivers.asm   — драйвер терминала: включение терминала и печать строк
os_main.asm          — главный цикл shell: вывод prompt, чтение команды, запуск команды
os_lib.asm           — библиотечные функции: strcmp, gets, atoi, itoa
os_strings.asm       — строки ОС и программ
programs.asm         — таблица программ и код пользовательских программ
README.md            — описание проекта
```

## Архитектура

### 1. Interrupt Vector Table

В начале памяти находится таблица векторов прерываний. Через неё процессор получает адреса обработчиков:

- старт ядра;
- обработчик по умолчанию;
- обработчик клавиатуры;
- системный обработчик.

### 2. Keyboard handler

Обработчик клавиатуры находится в `kernel_main.asm`.
Он не запускает программы напрямую. Его задача — только собрать строку ввода:

- читать символ из порта клавиатуры;
- сохранить символ в `kb_buffer`;
- обработать `Enter`;
- поставить флаг `kb_line_ready`;
- сбросить состояние клавиатуры через `kb_next`.

Такой подход важен, потому что ввод теперь может использовать не только shell, но и пользовательские программы.

### 3. `os_lib_gets`

В `os_lib.asm` реализована функция строкового ввода.

Условный аналог на C:

```c
fgets(buffer, size, stdin);
```

В ассемблерной версии:

```asm
r0 = адрес буфера назначения
r1 = максимальная длина буфера
jsr os_lib_gets
```

Функция ждёт, пока пользователь нажмёт `Enter`, затем копирует строку из общего клавиатурного буфера в буфер программы и завершает строку нулём.

### 4. Shell loop

Главный цикл находится в `os_main.asm`.

Логика работы:

1. включить терминал;
2. вывести приветствие;
3. вывести приглашение `>`;
4. прочитать строку командой `os_lib_gets`;
5. передать строку в `key_execute_command`;
6. снова вернуться к пункту 3.

### 5. Таблица программ

В `programs.asm` есть простая таблица программ — аналог очень простой файловой системы.

Идея:

```text
имя программы -> адрес кода программы
```

Когда пользователь вводит команду, ОС сравнивает введённую строку с именами в таблице. Если имя найдено, ОС получает адрес программы и делает переход к этому адресу.

Это позволяет добавлять новые программы без переписывания основного shell.

## Memory-mapped I/O

Проект использует устройства через фиксированные адреса памяти.

```text
0xF000 — порт символа клавиатуры
0xF002 — статус / управление клавиатурой
0xF008 — порт символа терминала
0xF00A — статус / управление терминалом
0xF010 — область таймера
```

Терминал и клавиатура работают через запись и чтение из этих адресов.

## Библиотечные функции

### `os_lib_strcmp`

Сравнивает две нуль-терминированные строки.

```asm
r0 = первая строка
r1 = вторая строка
r2 = 0, если строки равны
r2 = 1, если строки разные
```

Используется shell для поиска команды в таблице программ.

### `os_lib_gets`

Читает строку пользователя.

```asm
r0 = буфер назначения
r1 = максимальная длина
```

Используется программами `input`, `calc` и самим shell.

### `os_lib_atoi`

Переводит строку с десятичным числом в число.

Пример:

```text
"123" -> 123
```

Используется в программе `calc`.

### `os_lib_itoa_u16`

Переводит число в строку.

Пример:

```text
42 -> "42"
```

Используется в программе `calc` для вывода результата.

## Текущий статус

На текущем этапе в системе уже есть:

- рабочий терминал;
- ввод команд;
- таблица программ;
- несколько программ в одном образе;
- библиотечный ввод строки;
- интерактивная программа `input`;
- интерактивный калькулятор `calc`;
- начальная работа с таймером пребывания.

Проект уже показывает не просто набор программ, а минимальную ОС-структуру: ядро, драйверы, shell, библиотеку и пользовательские программы.

## Следующая задача: диспетчер задач

Следующий крупный этап — реализация диспетчера задач.

Планируемая идея:

- хранить список задач / программ;
- добавить структуру описания задачи;
- сохранять состояние задачи;
- переключать выполнение между задачами;
- использовать таймер как основу для переключения;
- сделать простую модель планирования.

Минимальная цель диспетчера задач:

```text
ОС должна не просто запускать одну программу по команде,
а уметь управлять несколькими задачами и переключать их выполнение.
```

## Как собрать

Из папки `src`:

```bash
cd ~/cdm-os/src
cocas *.asm -t cdm16
```

После успешной сборки появится образ, который можно загрузить в память симулятора.

## Как запустить в Logisim

1. Собрать проект через `cocas`.
2. Открыть схему в Logisim.
3. Загрузить полученный образ в RAM.
4. Запустить симуляцию.
5. Выбрать ввод с клавиатуры.
6. В терминале ввести команду, например:

```text
ls
input
calc
```
